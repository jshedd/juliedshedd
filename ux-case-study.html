User Experience Case Study:
Mississippi State University Libraries Website Redesign

Problem Statement

The MSU Libraries website had an outdated look and feel, was text- and jargon-heavy, and received negative feedback from students, faculty, and staff.

Principal Investigator

  • Julie D. Shedd, Web Services Specialist, Mississippi State University Libraries (Web Services Department)
  • Assistance provided by Web Services student assistants

Study Objectives

  • To learn more about the users of the MSU Libraries website
  • To present data-driven solutions to problems discovered through user feedback
  • To build a robust content infrastructure that enabled both users and website developers to find (and/or fix) what they needed quickly and easily

Process

Three usability tests - a card-sorting activity, a participatory design activity, and a contextual task analysis) were performed, with visitors to the Mitchell Memorial Library serving as study subjects. Subjects were recruited as they entered the Library and offered an incentive (a snack or drink) in return for their participation. Participants in each task were given a brief survey to collect some basic demographics and information about their use of the Libraries website. Results were reported to the MSU Libraries' Web Services Department, Assessment Committee, and Library Technologies Committee, and published on a website detailing the Libraries' self-assessment activities.

Activities

Card Sorting Task

A table was set with large cards labeled "Research," "Services," "About," "Need Help?," and "Other." 3x5" index cards were numbered and labeled with page titles from the MSU Libraries website. Participants were asked to examine these labels and sort them into the five main categories, arranging them in order of importance.

Participatory Design Task

Images of visual elements of the Libraries homepage were printed onto cardstock. Participants were asked to physically arrange these against a background to construct their ideal MSU Libraries' homepage, thinking out loud as they worked.

Contextual Task Analysis

A table was set with laptops and cameras. Participants were asked to perform three tasks on the Libraries homepage, thinking out loud as they worked. During this task, careful notes were taken, and participants' actions and facial expressions were recorded. Participants were then asked to complete a survey to indicate their feelings about the tasks, including how easy it was to find items, how intuitive the links were, and whether they found useful information. Feelings were assessed using a modified version of the System Usability Scale.
